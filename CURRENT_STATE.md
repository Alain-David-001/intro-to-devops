## Implementation details

- Branching strategy: trunk-based development. `main` is the trunk and should stay runnable; use short-lived branches for focused homework changes and PR/CI checks.
- Stack: Python 3.12 + FastAPI + Uvicorn.
- Runtime dependencies are pinned in `requirements.txt`; dev/test dependencies are pinned in `requirements-dev.txt`.
- Docker image definition lives in `Dockerfile`; build context exclusions live in `.dockerignore`.
- Application code lives under `app/`, as required.
- Root entrypoint is `main.py`, which creates the FastAPI app and includes the API router.
- Default storage is in-memory for local development and unit tests.
- MySQL storage is selected with `FRUITAPI_STORE=mysql` and reads database credentials from `DB_HOST`, `DB_PORT`, `DB_NAME`, `DB_USER`, and `DB_PASSWORD`.
- Terraform configuration lives in `infra/terraform` and creates VPC networking, ECS Fargate, RDS MySQL, Secrets Manager, security groups, IAM role, CloudWatch container logs, and an Application Load Balancer.
- RDS is not publicly accessible. MySQL port `3306` is only open from the ECS task security group.
- Public FruitAPI access is restricted by Terraform variable `allowed_cidr` on the ALB. ECS task port `8000` is only reachable from the ALB security group.
- Database password is generated by Terraform and stored in AWS Secrets Manager using a unique `name_prefix`, avoiding conflicts with secrets that AWS keeps scheduled for deletion. It is injected into ECS as a secret, not hardcoded in the repo.
- Terraform state and `terraform.tfvars` are ignored by Git and must not be committed.
- Release version source: `VERSION` currently contains `0.5.0`.
- Current HW5 image used by Terraform: `ghcr.io/alain-david-001/fruitapi:0.5.0`.

## Current API surface

- `GET /health` returns `{"status": "ok"}`.
- `GET /fruits` lists fruits. Optional query params: `in_season=true|false` and `limit=1` or larger.
- `POST /fruits` creates a fruit and returns it with an `id`.
- `GET /fruits/cheapest` returns the cheapest fruit or 404 if the store is empty.
- `GET /fruits/{id}` returns one fruit or 404.
- `PUT /fruits/{id}` updates provided fruit fields or returns 404.
- `DELETE /fruits/{id}` deletes a fruit and returns 204, or 404 if missing.

## Reproducibility notes

- Another user can clone the repo, configure their own AWS credentials, copy `infra/terraform/terraform.tfvars.example` to `terraform.tfvars`, set their own `allowed_cidr`, and run `terraform init`, `terraform plan`, and `terraform apply` to deploy their own copy.
- They should use the configured image or publish their own image and set `container_image` accordingly.
- They must not commit `terraform.tfvars` or Terraform state files.
- AWS resources can create costs. Run `terraform destroy` after finishing the homework/demo if the deployment is no longer needed.
