# HW4 Terraform

This Terraform configuration deploys the mandatory HW4 AWS resources for FruitAPI:

- VPC with two public subnets.
- RDS MySQL database.
- Secrets Manager secret for the database password.
- ECS Fargate cluster, task definition, and service running FruitAPI.
- Security groups so ECS can reach MySQL and only your configured IP can reach the FruitAPI task on port 8000.

## Prerequisites

- Terraform installed locally.
- AWS CLI credentials configured for a free-tier AWS account.
- The FruitAPI image already published to GHCR, e.g. `ghcr.io/alain-david-001/fruitapi:0.4.0`.

## Usage

```bash
cd infra/terraform
cp terraform.tfvars.example terraform.tfvars
# Edit allowed_cidr in terraform.tfvars to your public IP with /32.
terraform init
terraform plan
terraform apply
```

The database password is generated by Terraform and stored in AWS Secrets Manager. It is passed to the ECS task as a secret. Do not commit `terraform.tfvars` or Terraform state files.

After apply, find the public IP of the running ECS task in the AWS Console or with AWS CLI, then open `http://<task-public-ip>:8000/health`.

Example AWS CLI lookup:

```bash
TASK_ARN=$(aws ecs list-tasks --cluster fruitapi-cluster --service-name fruitapi-service --region eu-central-1 --query 'taskArns[0]' --output text)
ENI_ID=$(aws ecs describe-tasks --cluster fruitapi-cluster --tasks "$TASK_ARN" --region eu-central-1 --query 'tasks[0].attachments[0].details[?name==`networkInterfaceId`].value | [0]' --output text)
aws ec2 describe-network-interfaces --network-interface-ids "$ENI_ID" --region eu-central-1 --query 'NetworkInterfaces[0].Association.PublicIp' --output text
```

## Security notes

- RDS is not publicly accessible.
- MySQL ingress is limited to the ECS task security group.
- FruitAPI HTTP ingress is limited to `allowed_cidr`.
- The database password is generated and stored in Secrets Manager.
- If your public IP changes, update `allowed_cidr` in `terraform.tfvars` and run `terraform apply` again.

## Cleanup

AWS resources can create costs. Remove the deployment when it is no longer needed:

```bash
terraform destroy
```
