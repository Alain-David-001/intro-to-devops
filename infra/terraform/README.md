# Terraform Deployment

This Terraform configuration deploys the mandatory AWS resources for FruitAPI through HW5:

- VPC with two public subnets.
- RDS MySQL database.
- Secrets Manager secret for the database password.
- ECS Fargate cluster, task definition, and service running FruitAPI with two replicas by default.
- Application Load Balancer that serves FruitAPI on port 80.
- CloudWatch log group for FruitAPI container logs.
- Security groups so ECS can reach MySQL, the ALB can reach ECS, and only your configured IP can reach the ALB.

## Prerequisites

- Terraform installed locally.
- AWS CLI credentials configured for a free-tier AWS account.
- A FruitAPI image published to GHCR. The HW5 default is `ghcr.io/alain-david-001/fruitapi:0.5.0` after the branch is merged to `main`.

## Usage

```bash
cd infra/terraform
cp terraform.tfvars.example terraform.tfvars
# Edit allowed_cidr in terraform.tfvars to your public IP with /32.
terraform init
terraform plan
terraform apply
```

The database password is generated by Terraform and stored in AWS Secrets Manager. It is passed to the ECS task as a secret. Do not commit `terraform.tfvars` or Terraform state files.

After apply, use the `fruitapi_url` output and open `<fruitapi_url>/health`.

Useful output command:

```bash
terraform output fruitapi_url
```

## Security notes

- RDS is not publicly accessible.
- MySQL ingress is limited to the ECS task security group.
- ECS task HTTP ingress is limited to the ALB security group.
- Public FruitAPI HTTP ingress is limited to `allowed_cidr` on the ALB.
- The database password is generated and stored in Secrets Manager.
- If your public IP changes, update `allowed_cidr` in `terraform.tfvars` and run `terraform apply` again.

## Cleanup

AWS resources can create costs. Remove the deployment when it is no longer needed:

```bash
terraform destroy
```
